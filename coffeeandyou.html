Short description

This is a project given as part of Skilvul #tech4Impact curriculum under Kampus Merdeka program batch 1. With this assignment, I learned to create a simple website that tries to solve other people's problems. The target users are visitors who need information on their destination cafe. The development process starts from the BRD outline, wireframing, and finally coding.

Outline BRD

I analyze what the application is about, what problem I want to try to solve, features, target users, timeline, business flow, and user flow. The idea was not conceivable at first, with me and my friends adjusted what could be made before we implemented the programming logic in it.

Link : BRD

Outline UI/UX

We don't start the UX process from scratch. There are no style tiles or prototyping, all we need is wireframes to help shape the structure of the website.

Link : Wireframe

Development

Here comes the stage where we created the actual website. My role here was as the coding lead. I handed over the difficult and easy parts according to the abilities that each of us had.

Link : Github

Conclusion

I had the opportunity to demonstrate the ability to create a website with a group and get feedback from a professional mentor in their field. The product we created is not fully functional as an end-to-end platform, but we are starting the first step to create a product in a structured way.
